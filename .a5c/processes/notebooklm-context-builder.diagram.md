## Key Design Decisions

1. **7 thematic documents** instead of raw file dumps вҖ” each covers one aspect of the project, optimized for NotebookLM's retrieval
2. **Quality gate** before upload вҖ” documents are reviewed and fixed automatically if below threshold
3. **User breakpoint** before upload вҖ” lets you inspect and approve the content
4. **Verification via actual queries** вҖ” tests that NotebookLM can actually answer questions, not just that sources exist
5. **Update pipe uses AST parsing** вҖ” safely analyzes Python modules without importing them
6. **Notebook registration** вҖ” future queries use `"rare-books-bot"` as shorthand
